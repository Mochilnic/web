In this article I want to talk about tooltips. Despite the fact that this function is quite often used to quickly find a turnkey solution proved to be quite problematic. I went through a couple dozen options, but none satisfied. Briefly about what were looking for:

  • Adaptability is perhaps the main requirement. By 2019, this has long been the standard, and rightly so. Many solutions are based solely on CSS. Well, that's good if adaptability isn't required. But we are not satisfied. Tips on "pure CSS" can not calculate the position of the element on the screen, and work on the basis of pre-defined styles. That is why, tooltips only on CSS can be clipped in the mobile version. In some solutions, the location of the "pop-up" is set explicitly, manually: through the attributes "data-" or even directly in javascript, which is very inconvenient.
  • Using only the jQuery library, nothing extra to pull.
  • The Design part should be based on CSS with a minimum set of styles. Everything else: fonts, colors, spacing and more we adapt ourselves to the current task.
